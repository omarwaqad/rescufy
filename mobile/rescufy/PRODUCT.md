# Rescufy Product Context

## Product Overview

Rescufy is a multilingual emergency response ecosystem that connects patients, ambulance drivers, hospitals, and emergency administrators through real-time communication and intelligent dispatching.

The platform is designed to reduce emergency response delays while removing language barriers between all participants in the emergency workflow.

Primary platforms:

* Flutter Mobile Application

    * Patient App
    * Ambulance Driver App

* React Web Dashboard

    * Hospital Dashboard
    * Admin Dashboard

Languages:

* Arabic (RTL)
* English (LTR)

---

## Core Mission

Help people receive emergency medical assistance as quickly as possible while providing responders with accurate location, medical context, and real-time communication.

---

## Primary Users

### Patient

Goals:

* Request emergency assistance quickly
* Share location automatically
* Track ambulance progress
* Communicate in Arabic or English
* Maintain medical profile information

Pain Points:

* Panic during emergencies
* Difficulty describing conditions
* Uncertainty about exact location
* Language barriers

---

### Ambulance Driver

Goals:

* Receive emergency requests instantly
* Understand patient condition quickly
* Navigate efficiently
* Update emergency progress
* Communicate through clear workflows

Pain Points:

* Missing medical information
* Delayed dispatch visibility
* Connectivity issues
* GPS inaccuracies

---

### Hospital

Goals:

* Prepare before patient arrival
* View medical information
* Manage capacity and resources
* Receive emergency updates

---

### Admin

Goals:

* Monitor operations
* Analyze response performance
* Manage hospitals and drivers
* Track emergency metrics

---

## Product Principles

1. Emergency First
   Every screen must support rapid decision making.

2. Clarity Over Decoration
   Information hierarchy is more important than visual effects.

3. High Stress Usability
   Users may be anxious, injured, or distracted.

4. Fast Scanning
   Critical information should be visible within 2 seconds.

5. Accessibility
   Large typography, strong contrast, clear actions.

6. Bilingual by Design
   Arabic and English are first-class experiences.

7. Mobile First
   Most critical workflows occur on mobile devices.

## Core Workflows

### Patient Flow

Request Help
→ Location Detection
→ Emergency Submission
→ Ambulance Assignment
→ Live Tracking
→ Hospital Arrival
→ Feedback

### Driver Flow

Receive Request
→ Review Case
→ Accept Request
→ Navigate
→ Arrive
→ Treatment Started
→ Completed

### Hospital Flow

Receive Case
→ Prepare Resources
→ Receive Patient
→ Create Report
